Showcased Projects

VigilSiddhi Encoder

Desktop App for Video Stream Management

  • **Key Features:** Manages video/audio streams with FFmpeg. Supports a wide range of input/output types including local files, HTTP/RTMP/RTP/UDP, and SRT (caller/listener), as well as direct YouTube and dummy data sources.
  • **FFmpeg Tools:** Utilizes FFmpeg for advanced tasks like encoding, transcoding, stream copying, resizing, HLS streaming, and two-pass encoding for quality optimization.
  • **ffprobe & Monitoring:** Integrates ffprobe for detailed media analysis (streams, codecs, format). Features real-time, color-coded indicators (Green = OK, Yellow = Warning, Red = Error, Blue = Info) for live monitoring.
  • **Configuration & Scalability:** Supports repeatable workflows via YAML/JSON configs, with a future roadmap including GPU acceleration, web/PWA UI, batch processing, and advanced hardware acceleration.

VigilSiddhi: Real-Time Monitoring System

Scalable Multi-Agent Monitoring Platform

  • **Architecture:** An extensible real-time monitoring and alarm management platform unifying devices from broadcast, IT, and OT infrastructures.
  • **Multi-Agent Monitoring:** Collects data via SNMP, traps, and APIs from diverse sources like IRDs, network switches, Windows/Linux servers, and HP iLOs.
  • **Data Pipeline:** A robust data flow from agents to Redis Streams for event flow, then processed and stored in Elasticsearch for state/analytics and MySQL for configuration.
  • **Real-Time & Alerts:** Delivers live updates via a WebSocket-based UI and sends critical event notifications via Telegram and Email alerts, ensuring proactive alerting and operational visibility.
  • **Future Roadmap:** Plans include role-based access control, containerized deployment (Docker/K8s), auto-remediation features, and a mobile UI with push notifications.

mycas_V2: Modular Python Application Framework

Scalable Python Application Framework

  • **Architecture:** A Python-based, modular framework for enterprise and cloud-native applications. It features distinct layers: API, Core Logic, Database, Authentication, and Integrations.
  • **Interconnectivity:** Designed with clear separation of concerns, where the API handles requests, the Core Logic processes them, and the Database stores information, all secured by a dedicated Authentication layer.
  • **Advantages:** Highly modular, secure, and extensible, making it easy to maintain, test, and adapt for various use cases.
  • **Future Scope:** The roadmap includes a migration to microservices, full support for Docker/Kubernetes, CI/CD integration, and the addition of AI/ML modules for advanced functionality.
